### 2. 인터넷 통신

> * 클라이언트 ==(**요청**)==>> 서버
> * 서버 ==(**응답**)==>> 클라이언트

-> 일대다, 다대일, 다대다 등 다양한 경우에 대응하기 위해 IP 개념 도입(주소 부여)



### 3. IP(인터넷 프로토콜)

- **IP** : 집 주소라고 생각하자
- IP **패킷** : 패킷 정보 = 택배박스라고 생각하자 
  - 전송 데이터, 출발지, 경유지, 중간 허브 등

- **클라이언트에서 패킷을 전달**
- **IP 프로토콜의 한계 = 배송사고**
  - 비연결성 : 받을 대상이 없거나 <u>철거한 집으로</u> 박스 전송
  - 비신뢰성 : 중간에 박스가 사라지면? / 순서대로 물건이 안오면?
  - 프로그램의 구분 : 다른 사람이 이미 살고 있는 이전 집주소로 배달했다면?
